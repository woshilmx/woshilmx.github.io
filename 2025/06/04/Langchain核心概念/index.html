LangChain 核心概念

1. 什么是 LangChain？

LangChain 是一个专为构建大语言模型 (LLM) 驱动应用的开发框架，目标是使得 LLM 更易集成、组合和部署到真实产品中。

2. LCEL（LangChain Expression Language）

LCEL 是 LangChain 的链式表达式语言，是函数式、可组合、类型安全的方式，用于构建 LLM 流水线。

特点：

  • 声明式链构建
  • 支持并行执行、管道处理
  • 每一步都是 Runnable，可以组合成复杂的链
  • 支持调试追踪（Tracing）

3. 什么是链（Chain）？

链 是多个步骤按顺序组合执行的对象，常用于以下处理流程：

  • Prompt 构建
  • 模型调用
  • 结果解析
  • 多模态输出/输入

链支持串行、并行组合，也支持嵌套链。

4. Runnable 体系总览

Runnable 是 LangChain 中一切“可运行组件”的基类，是 LCEL 的核心构建块。

常见类型及用途：

类别 类名 作用
通用 RunnableLambda 将任意 Python 函数包装为链组件
透传 RunnablePassthrough 原样返回输入，常用于调试或结构保持
并行 RunnableParallel 并行运行多个 Runnable，返回 dict 聚合结果
串行 ` ` 运算符
组合链 RunnableMap 对输入集合每项运行同一 Runnable，类似 map()
输入分发 RunnableBranch 条件执行不同链
模型接口 ChatOpenAI, ChatAnthropic, etc. 封装不同厂商的大模型接口
解析器 StructuredOutputParser 用于结构化输出解析
Prompt ChatPromptTemplate 等 构建多轮对话 Prompt 模板

5. 串行链构建

使用 | 管道操作符组合步骤：

每一步都是 Runnable，例如：

  • prompt: 将输入格式化为消息模板
  • llm: 调用语言模型
  • output_parser: 结构化解析结果

6. RunnableLambda 用法详解

适用于：

  • 自定义前后处理逻辑
  • 手动构建规则逻辑
  • 灵活地插入数据操作步骤

7. RunnableParallel 并行处理

适用于：

  • 同时运行多个模型 / prompt 方案
  • 投票式选择方案
  • 性能优化并行推理

8. 实用链式组合示例

说明：

  • summary_chain: 获取功能摘要
  • RunnableParallel: 将摘要与模版传入下游链
  • recommend_chain: 模型推荐逻辑
  • triple_recommend: 多次调用 LLM，增强稳定性
  • combined_chain: 组合以上逻辑执行

9. ConsoleCallbackHandler 用法

用于在运行时打印每一步的输入和输出，方便调试链路。

10. LangChain 模块拆分（核心模块）

  • langchain_core: 包含所有轻量、无模型依赖的核心组件
  • langchain: 扩展组件，支持工具、数据库、agent 等
  • langchain_openai: OpenAI 模型接入封装
  • langchain_community: 社区贡献的连接器、工具

11. 组合技巧与最佳实践

  • 使用 partial() 预注入常量参数
  • 合理使用 RunnableLambda 将原生 Python 融入链中
  • 使用 StructuredOutputParser 保证输出结构可靠性
  • 使用 RunnableParallel 提升吞吐/多路径思维能力
  • 用 with_config(callbacks=[...]) 进行链级调试

RunnableBranch

简介

RunnableBranch 是 LangChain 提供的条件逻辑执行组件。它根据输入动态选择对应的 Runnable（即链）来执行，类似传统编程语言中的 if-elif-else 分支结构。

使用场景

适用于以下典型场景：

  • 根据用户输入类型决定执行哪一条链
  • 多意图识别处理：问答、翻译、摘要等
  • 模型/Prompt 的动态路由
  • fallback 降级链执行（提供默认处理逻辑）

基本用法

结构说明

  • 条件函数：接收一个输入参数 x，返回 True / False
  • runnable：任意实现 Runnable 接口的组件（如 RunnableLambda, ChatOpenAI 等）
  • 默认分支：当所有条件都不满足时执行

进阶示例：意图分类路由

注意事项

  • 条件函数按顺序匹配，匹配到第一个 True 的分支即执行，不会继续匹配。
  • 建议始终添加一个默认分支作为 fallback，避免未匹配时报错。
  • 可以嵌套使用 RunnableBranch 构建更复杂的决策树逻辑。
  • 可结合 RunnableParallel 实现条件 + 并行执行策略。

典型搭配组件

组件 用途说明
RunnableLambda 编写自定义处理逻辑
ChatOpenAI 等模型链 用于根据条件调用不同模型或 prompt
RunnablePassthrough 无修改地返回输入，常用于默认逻辑
RunnableParallel 用于在分支中并行执行多个处理链
